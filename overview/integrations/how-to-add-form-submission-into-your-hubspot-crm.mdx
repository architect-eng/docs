## Part 1: Create and Configure the Legacy App

<Steps>
  <Step title="Log in to HubSpot">
    Navigate to **Development > Legacy Apps**.
  </Step>

  <Step title="Create a Legacy App">
    Click **Create Legacy App** and choose the **Private** option. Set the app name to **Architect**.
  </Step>

  <Step title="Add permissions">
    Under the **Scope** section, add the following permissions:

    - `crm.objects.contacts.write`
    - `crm.objects.contacts.read`
  </Step>
</Steps>

## Part 2: Retrieve Your Access Token

<Steps>
  <Step title="Open the Auth tab">
    Go to the **Auth** tab within your new Legacy App.
  </Step>

  <Step title="Copy your token">
    Locate the **Access Token** section, click **Show Token** to reveal it, and copy it. You will use this as a Bearer Token in the `Authorization` header for all your HTTP requests.
  </Step>
</Steps>

## Part 3: Set Up Clay

<Steps>
  <Step title="Create a new table in Clay">
    Go to Clay and create a new custom table.
  </Step>

  <Step title="Add a Webhook column">
    Click **Add Column > Add Enrichment > Webhook**. Copy the Webhook URL from Clay.
  </Step>

  <Step title="Connect Architect to Clay">
    Go to the Architect platform and create a new webhook. Paste the Clay Webhook URL. Send a test example to make sure the connection works.
  </Step>

  <Step title="Add an HTTP API column">
    Go back to Clay and add a new column: **Add Column > Add Enrichment > HTTP API**.
  </Step>

  <Step title="Create an HTTP API account in Clay">
    Select **HTTP API (Headers)** account and create a new one in Clay.
  </Step>

  <Step title="Configure the HTTP request">
    Set up the following inputs:

    - **Method:** `POST`
    - **Endpoint:** `https://api.hubapi.com/crm/v3/objects/contacts`
    - **Body:**

    ```json
    {
      "properties": {
        "email": "email",
        "firstname": "firstName",
        "lastname": "lastName",
        "company": "companyName",
        "hs_google_click_id": "gclid"
      }
    }
    ```

    For each of the fields in the body, select the corresponding properties from the webhook data.

    <Note>
    If your form only captures emails, the body would look like:

    ```json
    {
      "properties": {
        "email": "email"
      }
    }
    ```
    </Note>
  </Step>

  <Step title="Set the headers">
    Add the following headers:

    | Header | Value |
    | --- | --- |
    | `Content-Type` | `application/json` |
    | `Authorization` | `Bearer YOUR_ACCESS_TOKEN` |

    <Warning>
    Replace `YOUR_ACCESS_TOKEN` with the token you copied from HubSpot in Part 2.
    </Warning>
  </Step>
</Steps>
