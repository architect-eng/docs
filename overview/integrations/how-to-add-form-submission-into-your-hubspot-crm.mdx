## Step 1: Add HubSpot Tracking to Architect Pages

Adding the HubSpot tracking script to your Architect pages allows HubSpot to drop its tracking cookie on visitors. This means when a visitor later submits a form, HubSpot can associate that submission with their full browsing activity and attribute the contact correctly.

<Steps>
  <Step title="Copy the HubSpot embed code">
    In HubSpot, go to **Settings**, then **Tracking & Analytics**, then **Tracking Code**. Copy the **Embed Code**. It will look something like this:

    ```html
    <!-- Start of HubSpot Embed Code -->
    <script type="text/javascript" id="hs-script-loader" async defer src="//js-eu1.hs-scripts.com/147460178.js"></script>
    <!-- End of HubSpot Embed Code -->
    ```
  </Step>

  <Step title="Add the script in Architect">
    Go to your Architect page, then navigate to **Settings** and click **Scripts**. Add a new script and paste the HubSpot embed code. Click **Save**.

    This ensures the HubSpot tracking cookie is loaded on your Architect pages so visitor activity is captured.
  </Step>
</Steps>

## Step 2: Connect a HubSpot Form

Choose the specific HubSpot form you want form submissions to be sent to, or create a new form in HubSpot if you don't have one yet.

<Steps>
  <Step title="Select or create a form in HubSpot">
    In HubSpot, go to **Marketing** and then **Forms**. Either select an existing form or create a new one using the HubSpot Forms API.
  </Step>

  <Step title="Connect the form in Architect">
    In Architect, link your CTA form submission to the HubSpot form so that leads are sent directly to HubSpot when visitors submit.
  </Step>
</Steps>

## Step 3: Create a HubSpot Legacy App

<Steps>
  <Step title="Log in to HubSpot">
    Go to **Development > Legacy Apps**.
  </Step>

  <Step title="Create a Private Legacy App">
    Name it **Architect** and add these scopes:

    - `crm.objects.contacts.write`
    - `crm.objects.contacts.read`
  </Step>

  <Step title="Copy your Access Token">
    Open the **Auth** tab, click **Show Token**, and copy it. This is your Bearer Token.
  </Step>
</Steps>

## Step 4: Set Up Clay

<Steps>
  <Step title="Create a table and add a Webhook">
    In Clay, create a new custom table. Add a column: **Add Enrichment > Webhook**. Copy the Webhook URL.
  </Step>

  <Step title="Connect Architect">
    In Architect, go to **Settings > Webhooks**, create a new webhook, and paste the Clay Webhook URL. Send a test to confirm it works.
  </Step>

  <Step title="Add an HTTP API column">
    Back in Clay, add a new column: **Add Enrichment > HTTP API**. Select **HTTP API (Headers)** and create a new account.
  </Step>

  <Step title="Configure the request">
    - **Method:** `POST`
    - **Endpoint:** `https://api.hubapi.com/crm/v3/objects/contacts`
    - **Body:**

    ```json
    {
      "properties": {
        "email": "email",
        "firstname": "firstName",
        "lastname": "lastName",
        "company": "companyName",
        "hs_google_click_id": "gclid"
      }
    }
    ```

    Map each field to the matching property from the webhook data.

    <Note>
    If your form only captures emails:

    ```json
    {
      "properties": {
        "email": "email"
      }
    }
    ```
    </Note>
  </Step>

  <Step title="Set the headers">
    | Header | Value |
    | --- | --- |
    | `Content-Type` | `application/json` |
    | `Authorization` | `Bearer YOUR_ACCESS_TOKEN` |

    <Warning>
    Replace `YOUR_ACCESS_TOKEN` with the token from Step 3.
    </Warning>
  </Step>
</Steps>

<Tip>
Architect webhooks also pass UTM parameters (such as `utm_source`, `utm_medium`, `utm_campaign`, and `gclid`) with each form submission, so you can map these to HubSpot contact properties for full attribution tracking.
</Tip>
