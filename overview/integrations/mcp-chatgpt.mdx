## Prerequisites

- A ChatGPT Plus, Pro, Business, or Enterprise account
- An Architect account with access to a workspace

## Setup

<Steps>
  <Step title="Open ChatGPT Integrations settings">
    Go to [chatgpt.com](https://chatgpt.com), click your profile icon, then navigate to **Settings** → **Apps & Connectors**.
  </Step>

  <Step title="Add Architect as a custom integration">
    Click **"+ Add Custom Integration"** and enter:

    - **Integration Name:** `Architect`
    - **Integration URL:** `https://mcp.tryarchitect.com/sse`

    Click **Save**.
  </Step>

  <Step title="Authenticate with Architect">
    ChatGPT will redirect you to sign in with your Architect account. Grant the requested permissions and your session will be saved.
  </Step>

  <Step title="Start using Architect tools">
    You can now ask ChatGPT to interact with your Architect workspace:

    - *"Search my Architect prospects for anyone at Acme Corp"*
    - *"List all my pages in Architect"*
    - *"Create a personalized landing page for John at acme.com"*

    <Note>
    Architect tools will appear in the integrations menu at the bottom of the chat input. You may need to select "Architect" from the integration picker for your first use.
    </Note>
  </Step>
</Steps>

## Alternative: Developer Mode

If you prefer more control, you can connect via Developer Mode:

<Steps>
  <Step title="Enable Developer Mode">
    Go to **Settings** → **Apps & Connectors** → **Advanced** → enable **Developer Mode**.
  </Step>

  <Step title="Add the MCP server">
    In the Developer Mode menu, add a new connector with the URL:

    ```
    https://mcp.tryarchitect.com/sse
    ```
  </Step>

  <Step title="Test the connection">
    Open a new chat and use the Developer Mode menu to select Architect tools. Try a simple prompt like *"List my Architect pages"* to verify the connection.
  </Step>
</Steps>

## Available tools

Once connected, ChatGPT has access to all Architect MCP tools:

| Tool | Description |
| --- | --- |
| `architect_person_get` | Get details for a specific person |
| `architect_person_search` | Search people in your workspace |
| `architect_person_create` | Create a new person with name and identifiers |
| `architect_person_append_log` | Add a log entry to a person's timeline |
| `architect_page_group_list` | List all pages in your workspace |
| `architect_audience_search` | Search audiences across pages |
| `architect_audience_get` | Get details for a specific audience |
| `architect_audience_create_for_person` | Create a person-linked audience |
| `architect_audience_get_url` | Get the published URL for an audience |
| `architect_audience_send_agent_message` | Send a message to the page agent |
| `architect_page_link_list` | List page links for your domains |
| `architect_page_link_create` | Create a new page link |
| `architect_page_link_get` | Get details for a page link |
| `architect_page_link_update_routing_table` | Update routing weights on a page link |
| `architect_page_link_delete` | Remove a page link |
| `architect_api_reference` | Browse the Architect Customer API spec |
| `architect_api` | Call any Customer API endpoint directly |

## Troubleshooting

<AccordionGroup>
  <Accordion title="I don't see 'Apps & Connectors' in settings">
    Custom integrations require a paid ChatGPT plan (Plus, Pro, Business, or Enterprise). Free accounts can't add MCP servers directly.
  </Accordion>

  <Accordion title="Authentication keeps failing">
    Try removing the integration and re-adding it. Make sure your Architect account has access to at least one workspace.
  </Accordion>

  <Accordion title="Tools aren't showing up in chat">
    Make sure you've selected Architect from the integrations picker at the bottom of the chat input. You may need to start a new chat after adding the integration.
  </Accordion>
</AccordionGroup>
