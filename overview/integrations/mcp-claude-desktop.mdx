## Prerequisites

- A Claude Desktop app installed ([download here](https://claude.ai/download))
- An Architect account with access to a workspace

## Setup

<Steps>
  <Step title="Open Claude Desktop settings">
    Open Claude Desktop and go to **Settings** → **Developer** → **Edit Config**.

    This opens the `claude_desktop_config.json` file in your text editor.
  </Step>

  <Step title="Add the Architect MCP server">
    Add the following to the `mcpServers` section of your config file:

    ```json
    {
      "mcpServers": {
        "architect": {
          "url": "https://mcp.tryarchitect.com/sse"
        }
      }
    }
    ```

    <Note>
    If you already have other MCP servers configured, add `"architect"` as a new key inside the existing `mcpServers` object.
    </Note>
  </Step>

  <Step title="Restart Claude Desktop">
    Close and reopen Claude Desktop for the changes to take effect.
  </Step>

  <Step title="Authenticate with Architect">
    The first time you use an Architect tool, Claude Desktop will open a browser window asking you to sign in to your Architect account. Once signed in, your session is saved and you won't need to sign in again.
  </Step>

  <Step title="Start using Architect tools">
    You can now ask Claude to interact with your Architect workspace. Try prompts like:

    - *"Search my prospects for anyone at Acme Corp"*
    - *"Create a personalized landing page for John at acme.com"*
    - *"What pages do I have in my workspace?"*
    - *"Send a message to the agent on my product demo page to update the headline"*
  </Step>
</Steps>

## Available tools

Once connected, Claude Desktop has access to all Architect MCP tools:

| Tool | Description |
| --- | --- |
| `architect_person_get` | Get details for a specific person |
| `architect_person_search` | Search people in your workspace |
| `architect_person_create` | Create a new person with name and identifiers |
| `architect_person_append_log` | Add a log entry to a person's timeline |
| `architect_page_group_list` | List all pages in your workspace |
| `architect_audience_search` | Search audiences across pages |
| `architect_audience_get` | Get details for a specific audience |
| `architect_audience_create_for_person` | Create a person-linked audience |
| `architect_audience_get_url` | Get the published URL for an audience |
| `architect_audience_send_agent_message` | Send a message to the page agent |
| `architect_page_link_list` | List page links for your domains |
| `architect_page_link_create` | Create a new page link |
| `architect_page_link_get` | Get details for a page link |
| `architect_page_link_update_routing_table` | Update routing weights on a page link |
| `architect_page_link_delete` | Remove a page link |
| `architect_api_reference` | Browse the Architect Customer API spec |
| `architect_api` | Call any Customer API endpoint directly |

## Example workflows

### Create a personalized page for a prospect

```
"Find the prospect John Smith at acme.com, then create a personalized 
audience for him on my Enterprise Demo page. Include context about 
their 500-person engineering team."
```

Claude will:
1. Search your prospects for John Smith
2. Find your Enterprise Demo page
3. Create a person-linked audience with the extra context
4. Return the published URL once the page is ready

### Bulk-update page copy

```
"Search for all audiences on my Product Launch page and send a message 
to the agent on each one to change the CTA from 'Get Started' to 
'Start Free Trial'."
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Claude says it can't find Architect tools">
    Make sure you've added the server config to `claude_desktop_config.json` and restarted Claude Desktop. Check that the URL is exactly `https://mcp.tryarchitect.com/sse`.
  </Accordion>

  <Accordion title="Authentication keeps failing">
    Try removing and re-adding the MCP server config, then restart Claude Desktop. If the issue persists, make sure your Architect account has access to at least one workspace.
  </Accordion>

  <Accordion title="Tools return 'missing required scope' errors">
    When you authenticate, make sure to grant all requested permissions. If you need to re-authorize, remove the Architect server from your config, restart Claude Desktop, re-add it, and authenticate again.
  </Accordion>
</AccordionGroup>
